# Domain Docs

This repo is a single-context repo. Engineering skills should read `CONTEXT.md` at the repo root and ADRs under `docs/adr/` before producing specs, issues, diagnostics, or implementation plans.

Use the glossary's vocabulary when naming domain concepts. If a proposed term conflicts with `CONTEXT.md`, prefer the glossary term or explicitly call out the conflict.

If output would contradict an ADR, surface the contradiction explicitly rather than silently overriding it.
